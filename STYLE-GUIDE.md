# Style guide (TaskFlow Help Center)

## Voice and tone
- Clear, direct, and user-focused
- Use active voice
- Avoid internal jargon

## Formatting standards
- Use sentence case headings (example: `## Create your first project`)
- Use numbered steps for procedures
- Use bullet lists for options and notes
- Use fenced code blocks only when needed

## UI conventions
- Use menu navigation like: `Settings > Workspace settings > Security`
- Button labels in bold: **Invite members**, **Save changes**
- Field names in backticks: `Workspace name`, `Email address`

## Troubleshooting writing
- State the symptom in user language
- Provide 2–4 likely causes
- Give specific actions the user can take

## Linking
- Use relative links within `/docs`
- Add “Related articles” at the end of articles where relevant
